# 智能机器人语音交互系统 — 系统说明

## 一、系统概述

本系统是一个面向智能机器人的**语音交互系统**，支持通过语音唤醒、声纹身份认证、语音指令识别、意图分类、MCP工具调用和多模态感知等能力，实现人机自然交互。系统可以区分用户的"闲聊"和"任务指令"两类意图，并分别做出对话回复或工具调用的响应。

### 核心能力

| 能力 | 说明 |
|---|---|
| 语音唤醒 | 通过持续监听麦克风，检测用户说出唤醒词（如"哈喽"）后激活系统 |
| 声纹识别 | 基于CAM++模型验证说话人身份，区分注册用户与陌生用户 |
| 语音识别 (ASR) | 基于Qwen3-ASR模型将用户语音转为文字 |
| 意图分类 | 基于文本向量化和余弦相似度，区分"闲聊"与"任务指令" |
| 工具选择 | 两层设计——先选能力类别，再通过function calling选择具体MCP工具 |
| 多模态感知 | 通过摄像头图像+多模态大模型识别用户（性别、年龄等），主动打招呼 |
| 对话记忆 | 短期记忆（最近N轮对话）+ 长期摘要（历史压缩）+ 用户偏好提取 |
| 语音合成 (TTS) | 基于Qwen3-TTS模型将文本回复转为语音输出 |
| 用户管理 | 新用户声纹注册、偏好记录、行为习惯持久化 |

## 四、各模块详细原理

### 4.1 语音唤醒模块 (`modules/wake_word_detector.py`)

#### 原理

系统没有使用专用的关键词检测（KWS）模型，而是采用**"录音→ASR→文本匹配"**的方案：持续录制2秒短音频片段，送入Qwen3-ASR模型识别，将识别结果与唤醒词列表做子串匹配。

#### 录音规则

| 参数 | 值 | 说明 |
|---|---|---|
| 采样率 | 16000 Hz | 标准语音采样率 |
| 声道 | 单声道 | 降低数据量 |
| 位深 | 16-bit PCM | PyAudio paInt16格式 |
| chunk大小 | 1024 帧 | 每次从麦克风读取的帧数 |
| 段时长 | 2.0 秒（固定） | 每段录音固定长度 |
| 总chunk数 | 31 个 | `int(16000/1024 × 2.0)` |

#### VAD（语音活动检测）

采用基于RMS音量的简易VAD，不使用神经网络VAD模型：

```
volume = L2范数(int16样本数据) / 样本数量
```

- `volume ≥ 8`（SILENCE_THRESHOLD）→ 该帧为"语音帧"
- `volume < 8` → 该帧为"静音帧"

#### 双重门控机制

为避免在纯静音环境下无谓调用ASR模型，设置两道关卡：

1. **最低语音帧数**：语音帧数 < 5（WAKE_MIN_VOICE_FRAMES）→ 丢弃该段
2. **最低语音占比**：语音帧占比 < 15%（WAKE_VOICE_RATIO_THRESHOLD）→ 丢弃该段

两道关卡都通过后才将音频送入ASR识别。

#### 唤醒词匹配

```python
WAKE_WORDS = ["哈喽", "哈楼", "哈咯", "halou", "ha lou", "hello"]
```

ASR识别文本转小写后，与每个唤醒词做**子串匹配**。设计多个变体是为了兼容ASR模型可能输出的不同同音字。

#### 主循环逻辑

```
while True:
    audio = record_2s_segment()
    if audio is None:          # 门控未通过
        continue
    text = ASR.transcribe(audio)
    if text contains wake_word:
        return True            # 唤醒成功
    # 否则继续循环
```

### 4.2 ASR用户语音录制模块 (`utils/audio_recorder.py`)

#### 原理

唤醒成功后，录制用户的实际指令/对话语音。采用**VAD驱动的动态录音**，等待用户开口后开始录音，检测到持续静音后自动停止。

#### 录音规则

| 参数 | 值 | 说明 |
|---|---|---|
| 采样率 | 16000 Hz | 与唤醒录音一致 |
| 声道 | 单声道 | |
| 位深 | 16-bit PCM | |
| chunk大小 | 1024 帧 | |
| 静音阈值 | 8 | 与唤醒录音共用 |
| 静音超时 | 1.5 秒 | 连续静音超过此时间则停止 |
| 最大录音时长 | 30 秒（硬上限） | 防止无限录音 |
| 静音帧上限 | 23 帧 | `int(1.5 × 16000 / 1024)` |

#### 两阶段录音流程

**阶段1 — 等待说话：**
```
持续读取chunk → 计算volume
  volume < 8  → 继续等待（不标记为已开始）
  volume ≥ 8  → speech_started = True → 进入阶段2
```

**阶段2 — 录音直到静音：**
```
持续读取chunk → 计算volume
  volume ≥ 8  → 语音帧，silent_frames归零
  volume < 8  → silent_frames += 1
  silent_frames ≥ 23 → 停止录音
```

如果全程未检测到语音（30秒内），返回None。

#### 首尾静音裁剪

录音结束后，自动裁剪前后静音段以减少ASR处理的无用数据：

```
从前往后扫描 → 找到第一个volume ≥ 8的帧 → start
从后往前扫描 → 找到第一个volume ≥ 8的帧 → end
两侧各加 0.1秒 padding（约1个chunk）
截取 frames[start : end+1]
```

#### 与唤醒录音的对比

| 维度 | 唤醒录音 | ASR用户录音 |
|---|---|---|
| 触发方式 | 待机时自动循环 | 唤醒成功后触发 |
| 录音时长 | 固定2.0秒 | 动态，最长30秒 |
| 停止条件 | 2秒录完 | 连续1.5秒静音 |
| VAD用途 | 过滤静音段避免无效ASR | 检测说话开始+结束 |
| 后处理 | 无 | 裁剪首尾静音+0.1秒padding |
| 输出文件 | `data/temp/wake_audio.wav` | `data/temp/live_audio.wav` |

### 4.3 ASR语音识别服务 (`services/asr_service.py`)

#### 原理

基于**Qwen3-ASR-0.6B**模型的语音识别服务，采用单例模式，预热后模型常驻GPU内存。

#### 模型配置

| 参数 | 值 |
|---|---|
| 模型 | Qwen3-ASR-0.6B |
| 精度 | bfloat16 |
| 设备 | cuda:0 |
| 最大批处理 | 32 |
| 最大生成token | 256 |

#### 工作流程

```
warm_up():  加载模型到GPU（启动时执行一次）
transcribe(audio_path):
    1. 读取WAV音频文件
    2. 模型推理 → 生成文本
    3. 返回识别结果
```

#### 设计要点

- **单例模式**：全局只有一个ASRService实例，避免重复加载模型
- **预热机制**：`warm_up()` 在系统启动时调用，后续 `transcribe()` 直接使用已加载的模型
- **流式接口预留**：`stream_transcribe()` 方法已定义但抛出 `NotImplementedError`，计划后续实现

### 4.4 声纹识别服务 (`services/voiceprint_service.py`)

#### 原理

基于ModelScope的**CAM++**（Speaker Verification Pipeline）声纹验证模型。将输入音频与声纹数据库中所有已注册音频逐一比对，返回相似度最高的匹配结果。

#### 模型配置

| 参数 | 值 |
|---|---|
| 模型 | CAM++ (campplus) |
| 任务 | speaker-verification |
| 匹配阈值 | 0.31 |

#### 识别流程

```
recognize(audio_path):
    best_user = None, best_score = 0.0
    for db_path, username in 声纹数据库:
        score = pipeline([audio_path, db_path])  # 1:1比对
        if score > best_score:
            best_score = score
            best_user = username
    if best_score ≥ 0.31:
        return (best_user, best_score)    # 匹配成功
    else:
        return (None, best_score)         # 未匹配
```

#### 声纹数据库

- 路径：`data/Authorized_User_Voiceprint_Database/`
- 格式：每个用户一个 `<用户名>.wav` 文件
- 注册时由用户语音录制并保存

#### 设计要点

- **单例模式**：模型加载一次，全局共享
- **数据库缓存**：启动时扫描目录下所有 `.wav` 文件，缓存路径和用户名映射
- **1:N识别**：遍历数据库中所有注册用户，取最高分且超过阈值的作为匹配结果

### 4.5 意图分类模块 (`modules/intent_classifier.py`)

#### 原理

基于**文本向量化 + 余弦相似度**的意图分类。使用Qwen3-Embedding-0.6B模型将文本编码为向量，通过计算用户输入与预定义指令库的相似度来区分"闲聊"和"任务指令"。

#### 向量化模型

| 参数 | 值 |
|---|---|
| 模型 | Qwen3-Embedding-0.6B |
| 库 | sentence-transformers |
| 相似度计算 | sklearn cosine_similarity |

#### 指令库

`command.txt` 包含 **153条** 预定义任务指令（跳过 `#` 开头的注释行），分为5大类：

| 类别 | 指令数 | 示例 |
|---|---|---|
| 状态查询 | 20条 | "机器人状态怎么样"、"看看电量还剩多少" |
| 空间记忆与场景理解 | 20条 | "我在哪里"、"记住这个地点" |
| 探索能力 | 20条 | "开始探索"、"自主建图" |
| 导航与局部运动能力 | 47条 | "往前走"、"前进三米"、"去厨房" |
| 任务控制能力 | 37条 | "确认执行"、"暂停任务"、"紧急停止" |

#### 分类流程

```
预热阶段:
    1. 加载 Qwen3-Embedding-0.6B 模型
    2. 读取 command.txt（跳过#注释行）
    3. 批量向量化153条指令 → command_embeddings
    4. 缓存到 data/command_embeddings.pkl（下次启动直接加载）

分类阶段:
    classify(user_text):
        1. 将用户文本向量化 → user_emb
        2. 计算 user_emb 与所有 command_embeddings 的余弦相似度
        3. 取最高相似度 max_sim
        4. max_sim ≥ 0.65 → 任务指令 (is_task = True)
        5. max_sim <  0.65 → 闲聊 (is_task = False)
        返回 (最匹配命令, max_sim, is_task)
```

#### 阈值设计

`INTENT_SIMILARITY_THRESHOLD = 0.65`

- 阈值越高 → 越倾向于判定为闲聊（更严格的任务判定）
- 阈值越低 → 越倾向于判定为任务指令（更宽松的任务判定）

### 4.6 MCP工具选择模块 (`modules/tool_selector.py`)

#### 原理

采用**两层设计**缩小选择范围，降低LLM决策难度：

```
第一层：能力类别选择（5选1）
    状态查询 / 空间记忆与场景理解 / 探索能力 / 导航与局部运动 / 任务控制

第二层：具体工具选择（该类别下的工具，使用function calling）
```

#### 第一层 — 能力类别选择

构建提示词列出5个能力类别及其描述，让LLM根据用户指令返回类别名称：

```
提示词: "请根据用户指令，判断属于哪种能力类别（只返回类别名称）"
输入: "前进三米"
输出: "导航与局部运动能力"
```

如果LLM返回的类别名不在预定义列表中，会尝试子串匹配校正，最终兜底为"状态查询"。

#### 第二层 — 具体工具选择

将能力类别下的工具定义转为**OpenAI function calling格式**，利用LLM的tool_choice能力自动选择：

```json
{
    "type": "function",
    "function": {
        "name": "relative_move",
        "description": "短距离相对运动，如前进三米",
        "parameters": {
            "type": "object",
            "properties": {
                "detail": {"type": "string", "description": "用户指令的详细描述"}
            },
            "required": ["detail"]
        }
    }
}
```

LLM返回：
```json
{"tool_name": "relative_move", "arguments": {"detail": "前进三米"}}
```

#### 能力与工具映射（定义在 `config.py` 的 `CAPABILITY_TYPES` 中）

| 能力类别 | 工具数 | 工具列表 |
|---|---|---|
| 状态查询 | 7 | get_robot_status, get_task_info, get_map_info, get_localization, get_sensor_status, get_planner_status, get_mcp_capabilities |
| 空间记忆与场景理解 | 5 | query_semantic_waypoint, get_exploration_summary, get_scene_state, get_panoramic_observation, write_semantic_memory |
| 探索能力 | 3 | start_exploration, stop_exploration, incremental_search |
| 导航与局部运动能力 | 5 | relative_move, navigate_to_pose, navigate_to_waypoint, follow_path, adjust_speed |
| 任务控制能力 | 5 | confirm_task, pause_task, resume_task, cancel_task, emergency_stop |

### 4.8 多模态感知模块 (`modules/multimodal_perception.py`)

#### 原理

利用多模态大模型（Qwen3.6-Flash支持图像输入）分析摄像头拍摄的图像，识别画面中的人物特征（性别、年龄、打扮），生成个性化的打招呼内容。

#### 工作流程

```
detect_and_greet(image_path):
    1. 将图像base64编码，构造多模态消息
    2. 发送给LLM，提示词要求：
       - 识别用户性别和大致年龄
       - 可以夸赞用户打扮
       - 询问有什么可以帮忙的
       - 控制在30字以内
    3. 返回 (是否检测到人, 打招呼文本)
```

#### 图像采集

由 `CameraStream`（`utils/camera_stream.py`）负责：

- 后台线程持续从摄像头读取帧（OpenCV VideoCapture）
- 唤醒后开启画面显示（`start_display`），录音结束后关闭
- `get_latest_frame()` 获取最新帧并保存为JPEG到 `data/temp/live_image.jpg`

### 4.9 对话记忆系统 (`modules/memory.py`)

#### 原理

实现三层记忆架构，使系统具备上下文理解能力：

```
┌─────────────────────────────────────────┐
│          对话记忆系统                     │
│                                         │
│  ┌─ 短期记忆 ─────────────────────────┐ │
│  │  最近 20 轮对话（MEMORY_MAX_TURNS） │ │
│  │  直接作为messages传给LLM            │ │
│  └────────────────────────────────────┘ │
│                                         │
│  ┌─ 长期摘要 ─────────────────────────┐ │
│  │  超过10轮后自动压缩旧对话为摘要     │ │
│  │  摘要以system消息注入              │ │
│  └────────────────────────────────────┘ │
│                                         │
│  ┌─ 用户偏好 ─────────────────────────┐ │
│  │  从对话中提取行为习惯和偏好          │ │
│  │  持久化到 <用户名>.md 文件          │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

#### 短期记忆

- 维护一个 `history` 列表，存储 `{"role": "user/assistant", "content": "..."}` 格式的对话轮次
- 构建消息时取最近 `MEMORY_MAX_TURNS`（20）轮

#### 长期摘要

- 当历史轮次超过 `MEMORY_SUMMARY_THRESHOLD`（10）且为5的倍数时触发压缩
- 将旧对话送入LLM，生成100字以内的摘要
- 摘要以 `system` 角色消息注入后续对话
- 多次压缩的摘要会拼接累积

#### 用户偏好提取

- 在每次交互完成后调用 `extract_preferences()`
- 将最近10轮对话送入LLM，提取JSON格式的行为习惯
- 提取结果通过 `UserManager.update_user_profile()` 持久化

### 4.12 TTS语音合成服务 (`services/tts_service.py`)

#### 原理

基于**Qwen3-TTS-12Hz-0.6B-CustomVoice**模型的语音合成服务，支持普通合成和流式合成。

#### 模型配置

| 参数 | 值 |
|---|---|
| 模型 | Qwen3-TTS-12Hz-0.6B-CustomVoice |
| 设备 | cuda:0 |
| 精度 | bfloat16 |
| 默认说话人 | Vivian |
| 默认语言 | Chinese |
| 输出采样率 | 24000 Hz |

#### 两种合成模式

**普通合成 `synthesize()`：**
```
输入: 完整文本
输出: 一次性合成整段语音 → WAV文件
```

**流式合成 `stream_synthesize()`：**
```
输入: LLM流式输出的文本生成器 (Generator)
处理:
    1. 逐token累积到句子缓冲区
    2. 遇到句末标点（。！？；. ! ? ; \n）→ 触发一次合成
    3. 所有句子合成完毕后拼接音频
输出: 合并后的WAV文件
```

流式合成设计使得TTS可以与LLM流式输出并行，减少端到端延迟。

## 五、服务层架构设计

### 5.1 单例模式

所有服务（ASR、声纹、LLM、TTS）均采用**单例模式**实现：

```python
class XXXService:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._initialized = False
        return cls._instance

    def __init__(self):
        if self._initialized:
            return
        # 初始化...
        self._initialized = True
```

**目的**：确保模型只加载一次，多个模块引用同一实例时不会重复加载。

### 5.2 预热机制

系统启动时通过 `Pipeline.warm_up_all()` 并行预热所有服务：

```python
targets = [
    self._voiceprint.warm_up,   # 加载CAM++模型
    self._llm.warm_up,          # 初始化OpenAI客户端
    self._asr.warm_up,          # 加载Qwen3-ASR到GPU
    self._tts.warm_up,          # 加载Qwen3-TTS到GPU
    self._intent.warm_up,       # 加载Embedding模型+预计算指令向量
]
threads = [threading.Thread(target=fn) for fn in targets]
# 5个线程并行启动
```

预热完成后，后续推理调用直接使用已加载的模型，避免首次调用的加载延迟。

### 5.3 并行处理

系统在两个关键位置使用多线程并行：

**并行点1 — 预热阶段：** 5个服务同时加载

**并行点2 — 声纹+ASR：** 对同一段用户语音同时执行声纹识别和语音识别

```python
t_vp = threading.Thread(target=do_voiceprint)   # 声纹识别
t_asr = threading.Thread(target=do_asr)         # ASR语音识别
t_vp.start(); t_asr.start()
t_vp.join();  t_asr.join()
```
